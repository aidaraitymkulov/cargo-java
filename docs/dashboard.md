# Дашборд (Dashboard)

## Обзор

Дашборд предоставляет агрегированную статистику для административной панели. Состоит из двух типов данных:
- **Карточки** — текущие числа (один запрос)
- **Графики** — динамика по дням за произвольный период (по одному запросу на каждый из 4 графиков)

Все эндпоинты доступны `MANAGER` и `SUPER_ADMIN`.

**Ограничение по роли:**
- `MANAGER` — всегда видит только данные своего филиала (параметр `branchId` игнорируется)
- `SUPER_ADMIN` — видит всё приложение; с `branchId` — конкретный филиал

---

## Карточки — GET /admin/dashboard/summary

**Query параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `branchId` | uuid | Опционально. Фильтр по филиалу (только для `SUPER_ADMIN`) |

**Response `200 OK`:**
```json
{
  "totalUsers": 312,
  "newUsersThisMonth": 24,
  "productsInChina": 120,
  "productsOnTheWay": 45,
  "productsAwaitingPickup": 23,
  "revenueThisWeek": 45000.00
}
```

| Поле | Описание |
|---|---|
| `totalUsers` | Все пользователи (с учётом фильтра по филиалу) |
| `newUsersThisMonth` | Зарегистрировались с начала текущего месяца |
| `productsInChina` | Товары со статусом `IN_CHINA` |
| `productsOnTheWay` | Товары со статусом `ON_THE_WAY` |
| `productsAwaitingPickup` | Товары со статусом `IN_KG` (прибыли в КР, ждут выдачи) |
| `revenueThisWeek` | Сумма `price` выданных товаров за последние 7 дней. **Только для `SUPER_ADMIN`** — для `MANAGER` всегда `null` |

---

## Графики

Все четыре графика возвращают одинаковый формат — массив ежедневных значений с нулями за дни без данных.

**Response `200 OK` (для всех графиков):**
```json
[
  { "date": "2026-05-01", "count": 5 },
  { "date": "2026-05-02", "count": 0 },
  { "date": "2026-05-03", "count": 8 }
]
```

**Общие query параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `from` | date (ISO) | Начало периода, включительно. Формат: `YYYY-MM-DD` |
| `to` | date (ISO) | Конец периода, включительно. Формат: `YYYY-MM-DD` |
| `branchId` | uuid | Опционально. Фильтр по филиалу (только для `SUPER_ADMIN`) |

---

### GET /admin/dashboard/charts/users

Количество новых зарегистрированных пользователей в каждый день периода.

Источник: `users.created_at`.

---

### GET /admin/dashboard/charts/products/in-china

Количество товаров, получивших статус `IN_CHINA` в каждый день периода.

Источник: `product_histories` где `status = IN_CHINA`.

---

### GET /admin/dashboard/charts/products/on-the-way

Количество товаров, получивших статус `ON_THE_WAY` в каждый день периода.

Источник: `product_histories` где `status = ON_THE_WAY`.

---

### GET /admin/dashboard/charts/products/delivered

Количество выданных товаров (`DELIVERED`) в каждый день периода.

Источник: `product_histories` где `status = DELIVERED`.

---

## Архитектурные решения

**Почему отдельные запросы для графиков, а не один?**
Фронтенд может обновлять каждый график независимо и задавать разные периоды для разных виджетов.

**Почему нули за дни без данных?**
Фронтенд не должен хранить логику заполнения пробелов — бэкенд всегда возвращает непрерывный массив за весь запрошенный период.

**Почему revenue только для SUPER_ADMIN?**
Бизнес-решение: менеджеры не должны видеть финансовую сводку.
