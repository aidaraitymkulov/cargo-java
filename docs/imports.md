# Импорт Excel (Imports)

## Обзор

Excel импорт — **запасной/ручной инструмент** для создания и обновления статусов товаров.  
Основной источник обновлений статусов — SmartPoint webhook (см. [smartpoint.md](smartpoint.md)).

Три эндпоинта соответствуют трём этапам логистики: регистрация в Китае, приёмка в КР, выдача клиенту.  
Все три принимают файл `.xlsx` как `multipart/form-data`, поле `file`.

**Принцип частичного успеха:** ошибка в одной строке/блоке не останавливает весь импорт — обрабатывается всё что можно, ошибки собираются в `errors[]`.

---

## Права доступа

| Роль | CN-импорт | KG-импорт |
|---|---|---|
| `MANAGER` | Любой файл (CN не привязан к филиалу) | Только файлы своего филиала |
| `SUPER_ADMIN` | Любой | Любой филиал |

---

## Формат ответа (все три эндпоинта)

```json
{
  "imported": 42,
  "errors": [
    {
      "row": 7,
      "reason": "USER_NOT_FOUND",
      "detail": "personalCode AN9999 не найден"
    }
  ]
}
```

## 1. Импорт из Китая

**`POST /admin/import/parcels/cn`**

Регистрирует товары прибывшие со склада в Китае. Каждая строка = один товар со статусом `IN_CHINA`.

### Формат файла

| Колонка A | Колонка B |
|---|---|
| `hatch` — трекинг-номер посылки | `personalCode` — код клиента |

**Пример:**

| A | B |
|---|---|
| YT123456789 | BSK0001 |
| JT987654321 | BSK0001 |
| YT111222333 | OSH0002 |
| YT555666777 | BIS0003 |

```
YT123456789  BSK0001
JT987654321  BSK0001   ← тот же клиент, другой hatch — норма
YT111222333  OSH0002
YT555666777  BIS0003
```

Строки где `hatch` или `personalCode` пустые — молча пропускаются.

### Алгоритм

1. Найти пользователя по `personalCode` → ошибка `USER_NOT_FOUND` если не найден
2. Создать `Product` со статусом `IN_CHINA`
3. Записать в `product_history`

### Особенности

- CN-файл общий для всех филиалов — ограничений по филиалу нет
- Дубликаты `hatch` у одного клиента — норма, создаётся отдельный товар
- Каждая строка — отдельная транзакция

## 2. Импорт после сортировки (in-kg)

**`POST /admin/import/parcels/kg/{prefix}/in-kg`**

Переводит товары в `IN_KG`, проставляет вес и цену на каждый товар.  
`{prefix}` — код филиала (`BSK`, `OSH`, `BIS`).

### Формат файла (блочный)

Файл разбит на блоки — один блок на одного клиента.

| Колонка A | Колонка B | Колонка C | Колонка D |
|---|---|---|---|
| `personalCode` (только на первой строке блока) | `hatch` — трекинг-номер | `weight` (кг) | `price` (сом) |

**Пример:**

| A | B | C | D |
|---|---|---|---|
| BSK0001 | YT123456789 | 1.25 | 850 |
| | JT987654321 | 0.80 | 420 |
| | YT111222999 | 0.50 | 300 |
| OSH0002 | YT111222333 | 2.10 | 1200 |
| BIS0003 | YT555666777 | 3.00 | 2000 |

```
BSK0001  YT123456789  1.25  850
         JT987654321  0.80  420    ← строки без personalCode привязываются к BSK0001
         YT111222999  0.50  300
OSH0002  YT111222333  2.10  1200
BIS0003  YT555666777  3.00  2000
```

### Алгоритм (на каждый товар в блоке)

1. Найти пользователя по `personalCode`
2. Для каждого `hatch`:
   - **Товар найден** (не DELIVERED) → обновить статус на `IN_KG`, проставить `weight` и `price`
   - **Товар не найден** → создать новый товар с `IN_KG`, `weight`, `price`
3. Записать в `product_history`

### Транзакции

Каждый блок — отдельная транзакция (`REQUIRES_NEW`). Ошибка одного блока не откатывает остальные.

## 3. Импорт выдачи (delivered)

**`POST /admin/import/parcels/kg/{prefix}/delivered`**

Переводит товары в `DELIVERED`. Формат файла **идентичен** `in-kg`.

### Пример файла

| A | B | C | D |
|---|---|---|---|
| BSK0001 | YT123456789 | 1.25 | 850 |
| | JT987654321 | 0.80 | 420 |
| OSH0002 | YT111222333 | 2.10 | 1200 |

### Алгоритм (на каждый товар в блоке)

1. Найти пользователя по `personalCode`
2. Для каждого `hatch`:
   - Найти товар со статусом `IN_KG` у этого пользователя (FIFO — самый старый)
   - Перевести в `DELIVERED`, обновить `weight` и `price`
   - Если товар не найден в `IN_KG` → ошибка `PRODUCT_NOT_FOUND`
3. Записать в `product_history`

---

## Соотношение Excel и SmartPoint

| Действие | SmartPoint webhook | Excel импорт |
|---|---|---|
| Создание товара `IN_CHINA` | ✓ AWAIT_POSTING | ✓ CN импорт |
| Перевод в `ON_THE_WAY` | ✓ ON_WAY | ✗ только SmartPoint или cron |
| Перевод в `IN_KG` | ✓ ON_PVZ | ✓ KG in-kg |
| Перевод в `DELIVERED` | ✓ ISSUED | ✓ KG delivered |

---

## Типичные ошибки импорта

| Код | Причина |
|---|---|
| `USER_NOT_FOUND` | `personalCode` не найден в системе |
| `BRANCH_NOT_FOUND` | Филиал с указанным `prefix` не существует |
| `FORBIDDEN` | Менеджер пытается загрузить файл чужого филиала |
| `PRODUCT_NOT_FOUND` | При `delivered` — товар со статусом `IN_KG` не найден |
| `IMPORT_ERROR` | Не удалось прочитать файл Excel |
