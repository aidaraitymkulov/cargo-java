# Домен: Новости (News)

Управление новостями приложения. Публичное чтение доступно без авторизации (Mobile API), создание и редактирование — только для сотрудников Admin API.

---

## Модель

### NewsResponse
```json
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "image": "/uploads/news/a1b2c3d4_photo.jpg",
  "title": "Новые рейсы из Гуанчжоу",
  "content": "С 1 июня добавляем дополнительный рейс...",
  "createdAt": "2026-05-21T10:30:00",
  "updatedAt": "2026-05-21T12:00:00"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | Уникальный идентификатор |
| `image` | string | URL изображения (`/uploads/news/{filename}`) |
| `title` | string | Заголовок новости |
| `content` | string | Текст новости |
| `createdAt` | ISO 8601 datetime | Дата создания |
| `updatedAt` | ISO 8601 datetime | Дата последнего обновления |

### PagedResponse (список новостей)
```json
{
  "items": [ NewsResponse, ... ],
  "page": 1,
  "pageSize": 20,
  "total": 42
}
```

## Mobile API — `/news`

Авторизация не требуется.

---

### GET /news — Список новостей

Возвращает постраничный список новостей, отсортированных по дате создания (новые первыми).

**Query-параметры**

| Параметр | Тип | По умолчанию | Ограничение |
|----------|-----|--------------|-------------|
| `page` | int | `1` | ≥ 1 |
| `pageSize` | int | `20` | ≥ 1 |

**Пример запроса**
```
GET /news?page=1&pageSize=10
```

**Ответ 200**
```json
{
  "items": [
    {
      "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "image": "/uploads/news/a1b2c3d4_photo.jpg",
      "title": "Новые рейсы из Гуанчжоу",
      "content": "С 1 июня добавляем дополнительный рейс...",
      "createdAt": "2026-05-21T10:30:00",
      "updatedAt": "2026-05-21T10:30:00"
    }
  ],
  "page": 1,
  "pageSize": 10,
  "total": 42
}
```

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 400 | `VALIDATION_ERROR` | `page` или `pageSize` меньше 1 |

---

### GET /news/{newsId} — Одна новость

**Path-параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `newsId` | UUID | ID новости |

**Ответ 200** — `NewsResponse`

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 404 | `NEWS_NOT_FOUND` | Новость с таким ID не существует |

## Admin API — `/admin/news`

Требуется авторизация через HttpOnly cookie (`X-Client-Type: web`).

| Операция | Роли |
|----------|------|
| GET (чтение) | `MANAGER`, `SUPER_ADMIN` |
| POST, PATCH, DELETE (запись) | только `SUPER_ADMIN` |

---

### GET /admin/news — Список новостей

Аналогично Mobile GET /news, но с аутентификацией.

**Query-параметры**

| Параметр | Тип | По умолчанию | Ограничение |
|----------|-----|--------------|-------------|
| `page` | int | `1` | ≥ 1 |
| `pageSize` | int | `20` | ≥ 1 |

**Ответ 200** — `PagedResponse<NewsResponse>`

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 400 | `VALIDATION_ERROR` | `page` или `pageSize` меньше 1 |
| 401 | `UNAUTHORIZED` | Нет cookie-сессии |
| 403 | `FORBIDDEN` | Недостаточно прав |

---

### GET /admin/news/{newsId} — Одна новость

**Ответ 200** — `NewsResponse`

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 401 | `UNAUTHORIZED` | — |
| 403 | `FORBIDDEN` | — |
| 404 | `NEWS_NOT_FOUND` | — |

---

### POST /admin/news — Создать новость

**Content-Type:** `multipart/form-data`

**Части запроса**

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| `title` | text | Да | Не пустой (NotBlank) |
| `content` | text | Да | Не пустой (NotBlank) |
| `image` | file | Да | MIME: `image/*`, размер: ≤ 20 MB |

**Пример запроса (curl)**
```bash
curl -X POST https://api.adesexpress.com/admin/news \
  -H "X-Client-Type: web" \
  --cookie "accessToken=..." \
  -F "title=Новые рейсы из Гуанчжоу" \
  -F "content=С 1 июня добавляем дополнительный рейс..." \
  -F "image=@/path/to/photo.jpg"
```

**Ответ 201** — `NewsResponse`

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 400 | `VALIDATION_ERROR` | `title` или `content` пустые / отсутствуют |
| 400 | `VALIDATION_ERROR` | Поле `image` отсутствует в запросе |
| 400 | `VALIDATION_ERROR` | MIME-тип файла не `image/*` |
| 401 | `UNAUTHORIZED` | — |
| 403 | `FORBIDDEN` | — |
| 413 | `FILE_TOO_LARGE` | Файл превышает 20 MB |
| 415 | `UNSUPPORTED_MEDIA_TYPE` | Content-Type не `multipart/form-data` |
| 500 | `IMAGE_UPLOAD_ERROR` | Ошибка записи файла на диск |

---

### PATCH /admin/news/{newsId} — Обновить новость

Все поля опциональны. Отправляйте только те, что нужно изменить.

**Content-Type:** `multipart/form-data`

**Path-параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `newsId` | UUID | ID новости |

**Части запроса**

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| `title` | text | Нет | Если передан — длина ≥ 1 (не пустая строка) |
| `content` | text | Нет | Если передан — длина ≥ 1 (не пустая строка) |
| `image` | file | Нет | Если передан — MIME: `image/*`, размер: ≤ 20 MB |

> При обновлении изображения старый файл удаляется с диска автоматически.

**Пример запроса (curl) — обновить только заголовок**
```bash
curl -X PATCH https://api.adesexpress.com/admin/news/3fa85f64-... \
  -H "X-Client-Type: web" \
  --cookie "accessToken=..." \
  -F "title=Обновлённый заголовок"
```

**Ответ 200** — `NewsResponse` (полный объект после обновления)

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 400 | `VALIDATION_ERROR` | `title` или `content` переданы как пустая строка |
| 400 | `VALIDATION_ERROR` | MIME-тип файла не `image/*` |
| 401 | `UNAUTHORIZED` | — |
| 403 | `FORBIDDEN` | — |
| 404 | `NEWS_NOT_FOUND` | Новость не найдена |
| 413 | `FILE_TOO_LARGE` | Файл превышает 20 MB |
| 415 | `UNSUPPORTED_MEDIA_TYPE` | Content-Type не `multipart/form-data` |
| 500 | `IMAGE_UPLOAD_ERROR` | Ошибка записи нового файла на диск |

---

### DELETE /admin/news/{newsId} — Удалить новость

Удаляет новость из БД и изображение с диска.

**Ответ 204** — тело отсутствует

**Ошибки**

| Код | error | Причина |
|-----|-------|---------|
| 401 | `UNAUTHORIZED` | — |
| 403 | `FORBIDDEN` | — |
| 404 | `NEWS_NOT_FOUND` | Новость не найдена |

## Загрузка изображений

### Ограничения
| Параметр | Значение |
|----------|----------|
| Максимальный размер файла | 20 MB |
| Максимальный размер запроса | 25 MB |
| Допустимые MIME-типы | `image/*` (jpg, png, webp, gif и т.д.) |

### Хранение
- Файлы сохраняются на диск по пути: `{upload-dir}/news/{uuid}_{safeName}`
- Имя файла санитизируется: запрещённые символы заменяются на `_`, path traversal (`../../`) обрезается
- URL в ответе: `/uploads/news/{filename}`
- Сервер раздаёт файлы статически по пути `/uploads/**`

### Безопасность при обновлении
При замене изображения (`PATCH` с новым `image`):
1. Новый файл записывается на диск
2. Новый путь сохраняется в БД
3. Старый файл удаляется с диска

Такой порядок гарантирует, что при ошибке записи нового файла старый остаётся нетронутым.

## Формат ошибок

Все ошибки возвращаются в едином формате:

```json
{
  "error": "ERROR_CODE",
  "message": "Человекочитаемое описание",
  "details": {
    "field": "описание конкретного поля"
  }
}
```

Поле `details` присутствует только у ошибок валидации (`VALIDATION_ERROR`).

### Пример ошибки валидации
```json
{
  "error": "VALIDATION_ERROR",
  "message": "Ошибка валидации",
  "details": {
    "title": "must not be blank",
    "content": "must not be blank"
  }
}
```

### Пример ошибки превышения размера
```json
{
  "error": "FILE_TOO_LARGE",
  "message": "Размер файла превышает допустимый лимит (20MB)"
}
```

### Пример ошибки отсутствия части
```json
{
  "error": "VALIDATION_ERROR",
  "message": "Отсутствует обязательная часть запроса: image"
}
```

## Пагинация

Все списковые эндпоинты поддерживают одинаковую пагинацию:

- `page` — номер страницы, начиная с `1` (обязательно ≥ 1)
- `pageSize` — количество элементов на странице (обязательно ≥ 1, по умолчанию `20`)
- Сортировка: по `createdAt` по убыванию (новые первыми)

---

## Коды ошибок

| error | HTTP | Описание |
|-------|------|----------|
| `NEWS_NOT_FOUND` | 404 | Новость с указанным ID не найдена |
| `VALIDATION_ERROR` | 400 | Нарушение правил валидации полей |
| `FILE_TOO_LARGE` | 413 | Размер файла превышает 20 MB |
| `UNSUPPORTED_MEDIA_TYPE` | 415 | Content-Type не `multipart/form-data` |
| `IMAGE_UPLOAD_ERROR` | 500 | Внутренняя ошибка при сохранении файла |
| `UNAUTHORIZED` | 401 | Отсутствует или истёк токен авторизации |
| `FORBIDDEN` | 403 | Недостаточно прав для выполнения операции |
